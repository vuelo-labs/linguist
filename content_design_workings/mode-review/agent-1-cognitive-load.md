# Agent 1 — Cognitive Load and Burnout Review
*Lens: where does the course create pressure, anxiety, or optimisation addiction?*
*Date: 2026-04-15*

---

## B-track: Module by Module

### B-01 — Your First Message

**Pressure vs permission:** Strongly permission-giving. The module is deliberately built to defuse anxiety. "Nothing you type here matters." "Nothing broke." This is the right register entirely. The objective itself ("know that nothing broke") is written as reassurance, not accomplishment. The optional second step and the explicit "you do not need to know anything about AI" framing are well-judged.

**Anxiety signals:** The "If it asks you to upgrade" section is pragmatically necessary but does introduce a mild note of scarcity (rate limits, resets, switching platforms). For a nervous learner this could feel like the tool is already working against them before they've even started. The tone is calm and factual, which mitigates this, but it's worth noting this is the first place where AI use has a resource ceiling named explicitly.

**Resting point:** Yes. The module ends with a single sentence pointing forward, not a list of things yet to learn. Feels complete as a standalone.

**Human Mode space:** Not present, and appropriately so for module one. The exercise is completely low-stakes exploration — "What are three things I could cook for dinner tonight?" as the personalised example is close to play, which is the right instinct at this stage.

**Does the exercise require knowing what you want?** No. The starter prompts are given. Nothing is demanded.

**Verdict:** Fine. No cognitive load concerns here. The most well-calibrated module in the B-track.

---

### B-02 — Copy, Personalise, Use

**Pressure vs permission:** The module is structurally sound, but it carries a subtle early-course pressure that compounds later. The template introduced here — with six labelled brackets — is immediately the most structured thing a beginner has seen. The instruction to fill every bracket correctly before sending creates a small performance frame: am I doing this right?

**Anxiety signals:** The line "what to leave out — what do you not want in the response? This is your chance to prevent things that would waste your time" introduces a negative efficiency frame early. The learner is now responsible not just for getting a response, but for pre-empting wasted time. For a confident learner, this is a useful skill. For a nervous one, it's a source of pre-send anxiety: what am I forgetting to exclude?

The relief valve is good: "If you are not sure what to put in a bracket — Guess." This is well-placed and well-worded. But it arrives after the full weight of the template has been presented.

**Resting point:** Adequate. The "What good looks like" section ends with "The goal is a message that fits your situation — not a message that matches the example above." That's reassuring. But the "Next" pointer immediately sets up B-04 as the logical next obligation, keeping momentum rather than allowing settling.

**Human Mode space:** Absent. The whole module is about structuring a command correctly. There's no moment where the learner is invited to wonder, explore, or not-know.

**Does the exercise require knowing what you want?** Yes, specifically. The template requires a role, a task, a verb, a specific thing, a length, and an exclusion. A learner without a task in mind will feel stuck at the "[brief description of task]" bracket. This is the first module that implicitly requires the learner to already have a clear output goal.

**Verdict:** Introduces the Machine Mode frame without naming it. The template is good pedagogy but the module doesn't acknowledge that sometimes you don't know what you want yet. That gap will matter more as the course progresses.

### B-03 — Platform Orientation

**Pressure vs permission:** No pressure. This is a reference module and correctly framed as such. The instruction to "Read only the section for the platform you are using" reduces scope anxiety.

**Anxiety signals:** None significant. One minor point: the note that B-09 will teach persistent instructions appears twice (in the Claude and ChatGPT sections), which subtly signals there is still more to set up. But this is mild.

**Resting point:** Good. "Come back here whenever you need to find something" is the right framing — it positions the module as a resource rather than a checkpoint.

**Human Mode space:** Not relevant here.

**Does the exercise require knowing what you want?** Not applicable.

**Verdict:** Fine. Reference module doing its job without adding pressure.

---

### B-04 — Your Input Shapes the Output

**Pressure vs permission:** This is the first module that introduces a comparison exercise (Version A vs Version B), and the framing lands in an interesting place. The message is: detailed input produces better output. The implicit corollary — undetailed input produces inferior output — creates a subtle before/after performance anxiety. The learner now has evidence that their previous, simpler messages were "worse."

The professional variant is particularly direct about this: "Which one could go directly into your report? Which one would you spend ten minutes editing?" This is useful for experienced professionals but introduces time-cost framing (ten minutes of editing = wasted time) that starts building the productivity optimisation mindset.

**Anxiety signals:** The "What is happening" section opens with a mirror metaphor: "The AI mirrors the level of detail you gave it." This is meant to be empowering, but it also means that every weak response is attributable to the learner's weak input. It's accurate, but it shifts responsibility entirely onto the user. A nervous learner will interpret every mediocre AI response as evidence of their own inadequacy rather than normal variance.

**Resting point:** The module ends by pointing to B-05 immediately. No settling.

**Human Mode space:** Absent. The exercise is entirely about producing better output more efficiently.

**Does the exercise require knowing what you want?** Yes — and in detail. The Version B template requires role, context, format, angle, and length. A learner without a real task will find this exercise abstract. The instruction to use "a paragraph from an email, an article, anything you have nearby" is pragmatic but doesn't account for learners who freeze when asked to produce real work material.

**Verdict:** The module normalises the idea that every interaction should be optimised for output quality. This is useful and accurate, but it's also the first significant step toward "always construct carefully, never just try." The Human Mode — just sending something to think with the AI — is not acknowledged as a valid use of AI here.

---

### B-05 — One Thing at a Time

**Pressure vs permission:** Functionally fine, but the opening line — "It feels efficient to ask for several things at once. In practice, it usually produces a response that does all of them slightly badly" — establishes a new performance standard: not just "does the AI answer?" but "is the AI giving this its full attention?" This introduces quality-of-attention as a metric the learner is now responsible for managing.

**Anxiety signals:** The three-message sequential pattern (read the summary, check it, then send message 2 after you are happy) creates a review loop at each step. This is good practice for focused tasks but, applied as a general habit, it becomes an exhausting quality-check routine. The exercise asks the learner to "read and check each response before sending the next" — which trains a monitoring posture, not an exploratory one.

**Resting point:** The "What good looks like" section ends with "The effort of sending three messages instead of one should be offset by spending less time editing the results." This is explicitly efficiency-framed. The rest point is actually a productivity justification.

**Human Mode space:** Absent. Entirely about execution efficiency.

**Does the exercise require knowing what you want?** Yes. The three sequential messages each require knowing exactly what to ask at each stage. There's no room in this pattern for "I'll send something and see what direction it goes."

**Verdict:** Continues the sequential tightening of the Machine Mode frame. The message "one task at a time = more control = less editing" is reasonable, but it's building a relationship to AI that is entirely about control and output quality, with no acknowledgment that sometimes sending something open-ended is the right tool.

---

### B-06 — Tell It When to Stop

**Pressure vs permission:** This is the clearest example of Machine Mode pedagogy in the B-track. The entire module is about constraining AI output — word counts, item counts, format limits, exclusions. Every mechanism taught here is a mechanism for tightening control. The learner is taught to think in boundaries before they send anything.

**Anxiety signals:** The framing "When you do not say when to stop, the AI has to guess. It usually guesses: more is safer than less" positions the AI's default behaviour as a problem to be corrected. The AI without constraints is the AI out of control. This is technically useful but creates an anxious pre-send checklist: did I set the right word count? Did I exclude the right things?

The "If this did not work" section begins: "If the AI went over your word count..." — which trains the learner to treat any length deviation as a failure state, and as their failure (they specified the condition too vaguely).

**Resting point:** None. The module ends with B-07 as the next thing to learn. The closing prompt ("Can you find the version that does exactly what I asked?") is progress-oriented.

**Human Mode space:** Absent. The mode demonstrated here is the purest Machine Mode in the course: task, verb, stopping condition. The exercise gives the learner no invitation to just see what the AI does when allowed to range freely, or to use that range productively.

**Does the exercise require knowing what you want?** Yes — with high specificity. Word counts, item counts, format exclusions. A learner who doesn't know yet what they want is blocked.

**Verdict:** Fine as a skill module. But the cumulative effect of B-02 through B-06 is that the learner is now running a pre-send mental checklist: role, task, verb, context, stopping condition, constraint. They haven't been taught that it's also fine to just ask. This is the module where the Machine Mode frame becomes load-bearing for the whole course.

### B-07 — What Just Happened? Reading AI Output

**Pressure vs permission:** The framing here is slightly better for the anxious learner — "The AI's first response is a draft. You are allowed to push back." The phrase "You are allowed to" is the right permission-giving language. The output review checklist, however, is a new ongoing obligation: four questions to work through "after every response, until it becomes automatic."

**Anxiety signals:** "Keep this checklist. You will use it for every module from here on." This is a significant ramp-up in ongoing cognitive load. The learner now has a standing review procedure they must apply to every AI response. For a learner doing one or two focused tasks a week, this is fine. For a learner who starts using AI extensively, this creates an obligation to evaluate before using, every single time.

The four checklist items include "Did it invent anything I did not give it?" — which, while important for accuracy, introduces a vigilance mode. The learner is now in a trust-but-verify relationship with AI output by default.

**Resting point:** Moderate. The "What good looks like" section ends with a clear decision tree (use it, ask for a change, or start over), which gives a sense of completion. But it also leaves the learner with an ongoing protocol rather than a skill they apply when needed.

**Human Mode space:** Absent. The checklist treats every AI interaction as a task with an output to be evaluated.

**Does the exercise require knowing what you want?** The module's exercise says "Send a real task — something from your actual work or life." This is the first explicit requirement that the learner bring real work. A learner without a clear task in mind will feel they're doing the module wrong.

**Verdict:** Introduces a healthy scepticism about AI output, which is genuinely valuable. But the "apply after every response" framing makes it an ongoing cognitive tax rather than a skill to reach for when something feels off.

---

### B-08 — Recovery

**Pressure vs permission:** Balanced. The intro is the right register: "The wrong response is to pile corrections on top of a bad start." This names the failure mode without shaming the learner for having been in it. The three recovery paths (correction, restate, new conversation) give concrete options without overwhelming.

**Anxiety signals:** The explanation of why layered corrections fail — "By the third or fourth correction, it may be as confused as it is capable of being" — is accurate and useful, but it also introduces a new category of AI failure: the confused AI. The learner now has to monitor whether the AI has accumulated too many conflicting signals, which is a meta-level attention demand.

**Resting point:** Good. This module does create a sense of "you can always start fresh and nothing is lost." The "The AI has no preference. Every new conversation starts completely fresh" framing is genuinely reassuring.

**Human Mode space:** Not present, but the "Help me write something" exercise that deliberately produces a vague result is actually one of the rare moments in the course where open-ended input is modelled — though only to demonstrate its inadequacy.

**Does the exercise require knowing what you want?** The restate template requires a specific task. But the module explicitly shows the learner what happens when you don't know (Step 1's "Help me write something"), and uses that as the entry point. This is a more honest acknowledgment of not-knowing than any other module in the B-track.

**Verdict:** One of the better modules for managing anxiety. The "start fresh" permission is important. But the module is still entirely Machine Mode in its prescriptions.

---

### B-09 — Your Two Projects

**Pressure vs permission:** This module introduces the terminology "Machine Mode" explicitly, which is significant. The template provided is labelled a "Machine Mode persistent instruction template" — the first time in the course the concept is named. This is also the only module in the entire B and E tracks that mentions Human Mode: "If you want to try Human Mode instructions as well — warmer, more conversational, less structured — that is a good next step. Set up a second project or Gem with different preferences."

This single mention is the entirety of Human Mode's presence in the beginner track. It appears in the "If this did not work" section, which means most learners will read it only if their persistent instructions failed. It is positioned as an alternative configuration for those who want it, not as an equally valid mode of working.

**Anxiety signals:** The template itself encodes efficiency anxiety: "No preamble. Ask me a clarifying question if my instruction is ambiguous — but only one question at a time. When you are done, stop. Do not summarise what you did or suggest next steps unless I ask." These are reasonable preferences, but they also teach the learner to suppress the AI's expansive, associative tendencies entirely. The AI is being trained (via the learner's instructions) to be maximally transactional.

**Resting point:** Adequate. The exercise is a genuine one-time setup with a clear done state.

**Human Mode space:** One sentence, in the troubleshooting section. This is the critical gap. The module names both modes but buries Human Mode as an optional add-on rather than treating it as a co-equal mode for different contexts.

**Does the exercise require knowing what you want?** The template requires the learner to specify a preferred length and their role, which is low-demand.

**Verdict:** This is the highest-leverage missed opportunity in the beginner track. Machine Mode is given a full template and a named module. Human Mode gets a single sentence in a troubleshooting section. A learner who reads this module carefully will set up Machine Mode defaults and may never think about Human Mode again.

---

### B-10 — The Opening Seed

**Pressure vs permission:** The Opening Seed is presented as the culminating skill of the beginner track, and its frame is completion-oriented ("getting it right takes about thirty seconds once you know what goes in it"). This is reassuring in intent but also implies that before this module, the learner was doing it less correctly. The "Close the loop" section at the end asks the learner to run a three-question diagnostic on every seed they write, which continues the ongoing evaluation obligation established in B-07.

**Anxiety signals:** "The more specific, the better. 'A draft' is vague. 'A 100-word draft email in a professional but warm tone, ending with a clear next step' is specific." This precision expectation, by B-10, is the eighth iteration of the same message: be specific, or your output will be worse. This is true, and it's important pedagogy, but the cumulative effect is that the learner has been told eight times that vagueness is a failure mode. The anxious learner will spend more time crafting the opening seed than doing the actual work.

**Resting point:** The stage completion prompt is well-designed — it acknowledges ten modules done and invites an email without forcing it. The "Ten modules down" message is the first genuine celebratory pause in the course. It works.

**Human Mode space:** Absent. The opening seed is a structure for maximally efficient task initiation. There's no mention of when to not use it — when you're just thinking with the AI, when you don't have a done condition, when you want to explore rather than execute.

**Does the exercise require knowing what you want?** Explicitly and by design. The four parts of the seed (role, task, context, done) all require a defined outcome. The module directly says "Think of a real task from your own work or life — something you actually need to do this week." A learner without a task is implicitly doing the module wrong.

**Verdict:** Fine as a synthesis module. The stage completion moment is the only genuine resting point in the B-track. But the cumulative effect of B-02 through B-10 is a learner who has learned to approach AI as a precision tool for known tasks. They have not been taught that it's also a thinking partner for unknown ones.

## E-track: Module by Module

### E-01 — The First Word Matters

**Pressure vs permission:** The threshold marker at the start of the E-track ("You can use AI. Now learn to control it") sets the entire stage's frame in its first line. Control is the explicit ambition. The module immediately introduces the concept of "risky verbs" — words that delegate judgement to the AI. The learner is now responsible for monitoring whether their word choices are safe or risky.

**Anxiety signals:** The verb categorisation (safe vs risky) creates a new pre-send evaluation: is my first word safe? The list of verbs to "use carefully" (Improve, Clean up, Make better, Enhance, Optimise) are common, natural words. Teaching the learner to treat them as hazards creates a linguistic vigilance that, if fully internalised, will affect how they write every AI message.

**Resting point:** The module ends with "Now that you can choose a verb that sets the right ceiling, E-02 shows you..." — no resting, direct forward momentum.

**Human Mode space:** Absent. The concept of a verb with "no ceiling" (improve, enhance) is treated as a problem. There's no acknowledgment that sometimes you want the AI to decide what "better" means — that open delegation can be productive when you don't yet know what you're looking for.

**Does the exercise require knowing what you want?** The "safe verb + stopping condition" pattern assumes you know the one specific change you want. Part 2 asks for a message "where it did more than you wanted" — which requires a history of specific-task AI use to draw on.

**Verdict:** The introduction of "risky verbs" is technically accurate but introduces a new category of learner-error that doesn't exist at the beginner level. The Human Mode insight — that sometimes you want the AI to range freely — is directly negated here without acknowledgment.

---

### E-02 — Tell It What to Leave Alone

**Pressure vs permission:** The intro is well-framed: "Prevention costs one line. Correction costs a whole extra conversation." This is a practical efficiency argument, not a fear argument. The constraint patterns are useful and concrete.

**Anxiety signals:** "The AI will do what you asked — and everything it thinks follows naturally from that." This continues the frame of AI as a system that will overreach unless constrained. The default state of AI is portrayed as expansive and needs to be contained. The learner is the container.

**Resting point:** The "If the AI ignored your constraint" section says "Do not patch it. Start a new message." This is clean and useful permission-giving for when things go wrong.

**Human Mode space:** Absent. The exercise is purely about limiting AI agency.

**Does the exercise require knowing what you want?** Yes — specifically, to know what you want left alone. This requires the learner to already have a sense of ownership over their work and a clear picture of what they don't want changed.

**Verdict:** Solid practical module. No significant new cognitive load introduced. The cumulative pressure is the concern, not anything specific here.

---

### E-03 — Context Is Fuel

**Pressure vs permission:** The "load-bearing test" introduced here is an elegant concept — ask whether removing a piece of context would change the output; if not, remove it. This is actually a pressure-reducing idea dressed up as a precision tool: it gives the learner permission to not over-explain.

**Anxiety signals:** The phrase "You do not need all of these for every task. You need the ones that would change the response if they were missing" is good — it limits the obligation. But the repeated emphasis on "relevant context" vs "decorative context" introduces another evaluation layer: is my context load-bearing or just noise? This is a useful question but it adds a pre-send audit step.

**Resting point:** The module ends with E-04 as the next step. No pause.

**Human Mode space:** Absent. The module continues the "fuel for a machine" metaphor — context is a resource to be optimised, not a way to open a conversation.

**Does the exercise require knowing what you want?** The context audit requires the learner to have a real opening seed from B-10. This is the first E-track module that explicitly builds on a prior exercise, which creates a dependency and can produce anxiety for learners who didn't complete or keep their B-10 seed.

**Verdict:** The load-bearing test is good thinking. The "context is fuel" metaphor is efficient but subtly frames every interaction as a fuel-loading operation before engine ignition. There's no model for context as starting a conversation.

---

### E-04 — Restate, Don't Patch

**Pressure vs permission:** This is the best module in the E-track for managing anxiety. "This is not failure. The AI has no memory of your effort. It does not register that you tried. Every new conversation is clean." This framing is genuinely liberating and is the closest the course comes to naming the low-stakes nature of AI interaction directly.

**Anxiety signals:** The three spiral signals ("your messages are getting longer, not shorter") are accurate and useful. But they add another meta-monitoring layer: the learner now has to watch for signs of their own correction spiral. This is a real risk pattern worth naming, but the cumulative monitoring load across B-07, B-08, and E-04 is significant.

**Resting point:** Good. The module has a clean conclusion: one message instead of six. The "The AI did not need to be corrected — it had what it needed from the start" framing is satisfying rather than anxiety-inducing.

**Human Mode space:** Absent.

**Does the exercise require knowing what you want?** The restate template requires high specificity. But the module is honest that this specificity comes from learning what failed — which is an experience-based rather than pre-knowledge-based model. This is a more honest framing than some earlier modules.

**Verdict:** One of the better modules. The "clean slate, no failure, start fresh" permission is valuable and underutilised elsewhere in the course.

---

### E-05 — Get the Plan First

**Pressure vs permission:** The two-message planning pattern is a practical tool for complex tasks, and the module is clear about when to use it and when to skip it ("Not every task needs a plan"). This is good scoping — it prevents the learner from applying the pattern universally.

**Anxiety signals:** "The point is not to correct it — the point is to catch a wrong direction before the AI has done any work" continues the AI-as-potential-source-of-rework framing. The learner is responsible for catching the AI's wrong turns before they become expensive. This is appropriate for the Advanced track but has a cost-of-failure quality that increases pre-session cognitive overhead.

**Resting point:** Adequate. The module ends with E-06 as the next step.

**Human Mode space:** The closest moment in the E-track: asking "how would you approach this?" before execution is adjacent to exploratory thinking. But it's still framed as a quality-control mechanism (catch the wrong approach early), not as a collaborative thinking moment.

**Does the exercise require knowing what you want?** Yes — the learner must identify a real multi-step task. But the module is realistic about this: "find one thing to adjust. It does not need to be a large change."

**Verdict:** Solid module. The "when to skip the plan" guidance prevents the pattern from becoming a ritual.

---

### E-06 — Migration and Re-seed

**Pressure vs permission:** The re-seed is framed as a skill for knowing when to move on, not as a failure recovery. "Starting fresh is not abandonment. The AI has no continuity preference." This is the second time in the E-track that the course explicitly defuses the anxiety of starting over. These two moments (E-04 and E-06) are the course's most direct engagements with learner anxiety.

**Anxiety signals:** The five re-seed signals (AI repeating corrected things, messages getting longer, task has shifted, working around mistakes, generic responses) create another ongoing monitoring checklist. These are the right signals to watch for, but after the B-07 checklist and the E-04 spiral signals, the learner is now running three distinct real-time monitoring routines in every significant AI session.

**Resting point:** Good. The comparison exercise (first response in the new conversation vs last few in the old one) has a clear done state.

**Human Mode space:** Absent. The re-seed is about carrying essential operational context forward, not about shifting modes.

**Does the exercise require knowing what you want?** The re-seed requires a one-sentence carry-forward and a new task — both require clear intent.

**Verdict:** Good module. The defusing of abandonment anxiety is valuable. But the number of monitoring signals the course has now given the learner is accumulating toward a meta-cognitive overhead that could exhaust careful learners.

---

### E-07 — Your Reference Card

**Pressure vs permission:** The opening is confident and well-calibrated: "After this, you do not need the course to use AI well." This is excellent — it positions the module as an exit door, not a gateway to more learning.

**Anxiety signals:** The three questions ("which three principles do you actually use?") invite reflection rather than performance. The instruction "This card is for you. No one else will read it" removes any external evaluation pressure. The section "What good looks like" ends with "You open it and immediately know what to send" — which is empowerment, not obligation.

**Resting point:** The stage completion prompt ("Seventeen modules down") is good. The closing keynote — "The machine will wait. It does not mind. Go do the work." — is the first direct permission to leave the learning environment and just work. This is exactly the right note.

**Human Mode space:** Still absent. The reference card synthesises only Machine Mode skills: verb shortlist, task templates, re-seed template, stopping conditions. A learner completing E-07 has a card for structured execution but nothing for exploratory use.

**Does the exercise require knowing what you want?** The three most common task types require the learner to have an established AI practice. A learner who just completed the beginner track and hasn't been using AI independently yet will find this exercise premature.

**Verdict:** The best closing module in the course. The reference card idea is pedagogically sound and the exit framing is exactly right. The only significant gap: the card, as designed, is a Machine Mode card. A learner who built this card would have no prompt for the times they don't know what they want.

## A-track: Module by Module

### A-00 — Getting Started with Claude Code

**Pressure vs permission:** The stage threshold ("From conversation to command line") is accurate but introduces an environment shift that could trigger anxiety in non-technical learners. The terminal basics section ("if you have only used GUI tools, the terminal can feel unfamiliar") acknowledges this but the remediation is practical rather than reassuring — it moves quickly to the four things you need to know rather than normalising the discomfort.

**Anxiety signals:** The Cost Preview section at the bottom — "Claude Code is not free. You are billed for usage" — is the first time in the entire course that AI use carries a direct financial cost. For learners who have been using free-tier chat interfaces through the whole B and E tracks, this is a significant frame shift. The instruction "Set a budget for yourself before your next session" adds a financial management obligation to the learning load.

**Resting point:** The "Your First Session" section ends with a clear success state (approved a tool call, received a response). This is well-structured. But the cost warning immediately follows the success moment and deflates it slightly.

**Human Mode space:** Absent. Claude Code is presented entirely through the lens of file operations, permissions, and tool calls.

**Does the exercise require knowing what you want?** The Copy-Personalise-Use template for A-00 is the standard opening seed plus a filename — lower demand than some E-track modules.

**Verdict:** The cost introduction creates a new category of anxiety not present in earlier tracks. Money is now in the frame.

---

### A-01 — Mental Model Refresher

**Pressure vs permission:** The three-part mental model is technically excellent and clearly written. But its effect on the anxious learner is to reveal how many new ways things can go wrong: the AI executes text not intent, the context degrades over time, and expensive tools can be selected by default. Every one of these is a new failure mode.

**Anxiety signals:** The Without/With pairs in this module are the most consequential cognitive load generators in the A-track. Each pair shows a plausible-looking but catastrophically wrong outcome (the developer whose payment error constraint vanished, the writer whose style rules drifted). These are accurate and instructive. They are also vivid illustrations of silent failure — things going wrong without the learner noticing until significant work is lost.

The phrase "Some details may not survive" (about compaction) is accurate but ominous for a learner who is now responsible for structuring their messages to survive a process they cannot observe or control.

**Resting point:** The three-part summary table is a good landing point. The module is intellectually well-organised.

**Human Mode space:** Absent. The mental model is entirely about control and precision.

**Does the exercise require knowing what you want?** The Copy-Personalise-Use template requires specific facts and constraints. Higher demand than B-track templates.

**Verdict:** Introduces the concept of invisible failure in long sessions. This is important knowledge but it adds a new vigilance layer. The learner is now monitoring for compaction loss as well as all the earlier signals.

---

### A-02 — Cost, Tokens, and What Things Actually Cost

**Pressure vs permission:** The opening is notably well-calibrated: "It is not here to make you cautious. It is here to make you informed." This is the right framing. The module demystifies cost rather than weaponising it.

**Anxiety signals:** The four cost drivers ("Long context," "Many tool calls," "Long sessions," "Agent delegation") each carry mitigation advice, which keeps the module practical rather than alarming. The worked example showing a nine-cent session is reassuring — it grounds the abstract ("tokens") in a concrete and small number.

However: the rough benchmarks section ends with "Agent delegation on a complex task: $5–20 depending on scope." For learners with limited income or tight budgets, this is not trivial. The learner now has a cognitive budget-tracking obligation on top of the task-quality obligations from earlier tracks.

**Resting point:** Good. The module ends with a clean "you now have the mental model" summary. It doesn't drive urgency.

**Human Mode space:** Not applicable to this module's subject matter.

**Does the exercise require knowing what you want?** Not significantly.

**Verdict:** The best-framed of the A-track modules for managing anxiety. The cost demystification is done carefully.

---

### A-03 — CLAUDE.md

**Pressure vs permission:** The setup exercise ("Write three to five rules") is well-scoped. The instruction to "Start small" and "Add rules as you notice patterns" prevents the learner from over-engineering the file before they have experience. The "Your first CLAUDE.md will be incomplete. That is expected" framing is permission-giving.

**Anxiety signals:** The Without/With examples show a developer who forgets to type their constraints and ends up with violations reaching code review. For non-technical learners who don't work in code review contexts, this example maps less directly. But the content-work example (style rules drifting mid-session) is clear and creates a reasonable anxiety about omission.

**Resting point:** Good. The module has a clear four-step exercise with a verification step. The instruction to "Iterate" accepts incompleteness from the start.

**Human Mode space:** The user-level CLAUDE.md example includes "explain your reasoning, don't just give me the answer" — this is the closest any A-track module comes to acknowledging that exploratory, reasoning-visible AI use has value. It's one line among many, but it's there.

**Does the exercise require knowing what you want?** The template requires known preferences, which assumes the learner has run enough sessions to have preferences. The instruction to "ask yourself: what do I find myself repeating in every session?" is pragmatic but requires a history.

**Verdict:** Solid, well-calibrated. The one line about explaining reasoning is a small but real acknowledgment of Human Mode.

---

### A-04 — Context Awareness and Compaction

**Pressure vs permission:** This is the module with the highest anxiety potential in the entire course. Compaction is invisible, it happens automatically, it can silently lose constraints and decisions, and the learner cannot directly observe or control when it fires. The list of what compaction loses is more detailed and scarier than the list of what it preserves.

**Anxiety signals:** The symptoms table ("Claude Code re-asks a question you already answered → that answer did not survive compaction") is technically excellent and genuinely important for advanced users. It's also a vivid catalogue of invisible failure. A learner who takes this seriously will feel the weight of monitoring for compaction loss on top of all other monitoring obligations.

The phrase "most AI users do not know this happens. Knowing it — and knowing what survives and what does not — is one of the things that separates an advanced practitioner from a capable beginner" is motivating, but it also implies that capable beginners are operating in the dark and losing work they don't know they're losing.

**Resting point:** The compaction survival checklist (five items) is a new standing protocol — the most demanding persistent obligation introduced in the course. It joins the B-07 output review checklist and the E-04 spiral signals as ongoing cognitive maintenance routines.

**Human Mode space:** Absent.

**Does the exercise require knowing what you want?** The exercise requires a real 6–10 turn task, which is the highest task demand in the course so far.

**Verdict:** Necessary knowledge, genuinely important. But this module adds the most new cognitive load of any module in the course. The accumulation of standing monitoring routines (B-07 checklist + E-04 spiral signals + A-04 compaction checklist) creates a maintenance overhead that would exhaust a careful learner who tried to run all three in parallel.

## Overall Pattern: The Cumulative Effect

A learner who follows this course carefully from B-01 to A-09 undergoes a coherent and deliberate curriculum. The individual modules are well-written, technically accurate, and often reassuring in isolation. The cumulative effect is a different thing.

**What the course builds:** a competent Machine Mode practitioner. By E-07, the learner can initiate any structured task, constrain the output, evaluate the response, recover from failure, migrate context, and personalise a reference card. By A-09, they can manage cost, survive compaction, scope sessions, plan before executing, direct tools, and brief subagents. These are genuinely valuable skills.

**What the course does not build:** any relationship to AI that isn't transactional and goal-directed. The learner has not been shown how to think with AI. They have not been shown how to use AI when they don't know what they want. They have not been given permission to use open-ended, exploratory prompts as a legitimate mode. The single exception — "never delegate understanding" in A-08 — names the distinction correctly but frames it as a delegation rule for advanced practitioners, not a mode available to beginners.

**The monitoring load problem:** By the end of the course, a careful learner is running four overlapping monitoring routines:
1. Output review checklist (B-07): four questions after every response
2. Correction spiral signals (E-04): three patterns to watch for in active conversations
3. Re-seed signals (E-06): five signals that a conversation has run its course
4. Compaction survival checklist (A-04): five practices for long sessions

Each routine is appropriate in its own module. Together, they create a cognitive maintenance overhead that can make extended AI use feel more like quality control than work. A learner who tries to run all four continuously will find AI use exhausting.

**The specificity trap:** From B-04 onward, the course consistently teaches that vagueness is a failure mode. This is accurate for execution tasks. But it creates an implicit message that the only valid AI use is specific-task execution. Learners who internalise this will feel they are doing AI wrong when they're using it to think, explore, or draft without a clear endpoint in mind. They will either force artificial specificity onto tasks that don't warrant it, or avoid using AI for those tasks altogether — missing the most generative use of the tool.

**The anxiety structure:** The course manages acute anxiety well (B-01's "nothing broke," E-04's "this is not failure," E-06's "starting fresh is not abandonment"). It manages chronic anxiety less well. Chronic anxiety is the cumulative effect of: being responsible for output quality (B-04), monitoring for AI failure modes (B-07), watching for correction spirals (E-04), maintaining session hygiene (A-05), surviving compaction (A-04), and managing financial cost (A-02). None of these creates a burnout risk individually. Together, they build a mode of AI engagement that is permanently alert and rarely at rest.

**The Human Mode gap:** The term "Machine Mode" appears once in B-09, where a template is explicitly labelled as "Machine Mode defaults." Human Mode is mentioned in the same module — once, in the troubleshooting section. This imbalance is not a content problem; it's a frame problem. The course's working assumption is that good AI use is precise, structured, and output-oriented. Human Mode — using AI to think, explore, not-know — is positioned as an optional personalisation, not as a mode the course actively teaches.

## Three Specific Recommendations

### Recommendation 1: Add a Human Mode module at B-05.5 (between B-05 and B-06)

The current course moves from "one thing at a time" (B-05) directly to "tell it when to stop" (B-06). Both are Machine Mode skills. Before the course teaches stopping conditions, it should teach a different mode entirely: using AI to think before you know what you want.

A module here — call it "B-05.5: Think With It First" or similar — would teach the learner that AI is useful before you have a task. Exercises might include:
- "I'm not sure what I think about [topic]. Talk me through the main considerations."
- "I have a vague sense that [situation]. What questions should I be asking?"
- "Here's a messy draft of something I'm working on. What's interesting here? What's missing?"

These prompts don't have stopping conditions. They don't have safe verbs. They produce unpredictable output that the learner reads for provocation, not for use. This mode should be named and validated before the course spends five more modules teaching its opposite.

The cognitive load benefit: learners who understand they have two modes — Machine Mode for execution, Human Mode for thinking — will stop applying Machine Mode discipline to situations that don't need it. This reduces the anxiety of the specificity requirement ("I don't know what I want yet, so I can't use AI well"). It gives the learner a map with two territories instead of one.

This is the highest-leverage single change in the course.

---

### Recommendation 2: Consolidate and gate the monitoring routines

The course currently teaches four separate monitoring routines across four modules (B-07, E-04, E-06, A-04). Each is appropriate in context. But a learner who reads all four carefully will feel that competent AI use requires continuous multi-layer monitoring — which is the psychological structure of performance anxiety.

The fix is not to remove the monitoring advice but to frame it explicitly as a tiered system:

**Default (B-track completion):** One simple question after every response. "Did this do what I asked?" That's it. Not four questions. One.

**When something feels off (E-track):** Then deploy the fuller checklist from B-07. The checklist is a diagnostic, not a routine.

**For long sessions (A-track):** Compaction and spiral monitoring are advanced skills for extended technical work, not everyday chat use.

Currently, all four monitoring systems are presented as ongoing obligations. Reframing them as a tiered response to increasing complexity would reduce the standing monitoring load and let ordinary learners use AI without running four mental processes per response.

---

### Recommendation 3: Surface the "never delegate understanding" principle early — as a mode statement, not a delegation rule

A-08 contains the most important single line in the course: "Never delegate understanding. If you need to understand something — to learn it, to develop a view on it, to make a decision based on it — do it in your main session."

This should be in B-09 or B-10, not A-08. It is not an advanced practitioner insight — it is a foundational orientation that should shape how every learner approaches AI from their first week.

The reframing for earlier placement: "There are two modes of using AI. Machine Mode is for tasks you understand well enough to define clearly. Human Mode is for tasks you need to think through first. You'll know you're in Human Mode when you don't have a done condition yet — when you're using the AI to help you figure out what done looks like."

This gives the learner permission for the open-ended, exploratory interactions that the rest of the course subtly trains them to avoid. It also provides an exit from the specificity trap: if you can't fill in the brackets, you might be in Human Mode — and that's fine. Start a different kind of conversation.

The burnout mechanism this prevents: learners who have only Machine Mode available will either force-fit every AI use into a structured prompt (exhausting) or stop using AI for anything they're uncertain about (underutilising the tool). Neither outcome serves them. The two-mode frame — introduced early and returned to — is the structural intervention that makes the rest of the course's precision skills sustainable.
