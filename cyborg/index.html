Veralux Analytics — Workspace Setup

A 7-day data analyst assessment inside a simulated company. The workspace runs locally on your machine via Docker, captures how you work, and submits when you're done.

What this is

A self-paced, simulated work week. You'll inherit the data environment of a fictional company, Veralux Analytics — databases, dashboards, Slack exports, ticket archives, git repos, stakeholder profiles. You'll have access to AI tools the same way you would on the job. There's no graded "right answer" — we want to see how you work with messy reality.

The exercise feeds an interview, not an algorithm. A human reads what you produce and how you got there.

What you'll need

  1. Your token — sent in your invitation email. Single-use.
  2. Docker Desktop — download here. Mac, Linux, or Windows-with-WSL2.
  3. ~5 GB of free disk for the workspace image, and a laptop that can run a local container without melting.
  4. Stable internet for the one-time AI sign-in (OAuth) and image pull. The container itself runs locally.
  5. An IDE you're comfortable with — VS Code or Cursor with the Remote-SSH extension is the smoothest path. The browser also has a built-in terminal and SQL runner if you'd rather stay there.
  6. About 5 minutes to install. Then 7 days of clock time to use the workspace however suits you.

To get the most out of it

Treat this like a first week at a new job, not a take-home with a stopwatch. People who do well usually:

  1. Block out one or two real working sessions. The environment rewards depth — context-switching every five minutes flattens the experience.
  2. Read before they act. The handover notes, stakeholder profiles, and recent Slack threads are there for a reason.
  3. Write some of their own SQL. The data is small enough to explore by hand. Don't outsource everything to the assistant.
  4. Fix friction when they hit it. The environment has some intentional rough edges; how you handle them is informative.
  5. Use the AI tool the way you actually use one at work. Don't perform — we'd rather see your real workflow than a curated one.
  6. Take notes as you go. By day three you'll have forgotten what you noticed on day one.

Activity is logged. Browser app opens, terminal commands, AI prompts and responses, file accesses, SQL queries, and edits to a small set of configuration files are all captured. This is so a human can understand how you worked, not so a model can score you. There are no trick questions and no penalty for things you didn't find — but be aware that "I worked it out without using the AI" is verifiable, and so is the opposite.

About the AI tooling

Claude Code is the supported, first-class assistant for this assessment. It's pre-installed in the workspace. On first run it asks you to sign in via OAuth — any free Anthropic account works, no API key required, no payment.

We chose Claude as the bundled tool for three reasons: (1) it's what the assessment author uses day-to-day, so it's the tool the environment was designed and tested against; (2) it's the assistant we get the highest-fidelity activity capture from, which feeds the human review of your work; and (3) zero setup means every candidate starts from the same line. First-class support for other vendors is coming — not in this launch.

You can use a different AI tool if you'd prefer — the workspace exposes SSH on port 2222 (user candidate, password = your token), and any tool that supports remote shells works:

  1. Cursor or VS Code — Remote-SSH extension → candidate@localhost:2222 → open /workspace. Use whichever in-IDE assistant you have set up (Cursor's built-in, Copilot, Continue, etc.).
  2. Terminal-based agents — open an SSH session and run aider, codex, gemini, or anything CLI-driven. You'll need to bring your own API key for those.
  3. Other GUI tools — anything that supports Remote-SSH or a remote workspace works the same way as Cursor/VS Code.

A note on fidelity: when you use Claude Code we capture which tools it called and what files it touched. With other assistants we capture the file changes and your terminal commands but not the AI's internal calls. Reviewers compensate for this — using a different tool isn't penalised — but it's worth knowing the picture is slightly less detailed.

Honest expectations

  1. This is unusual. Most processes give you a take-home or a whiteboard. We're giving you a week inside a messy environment because the job is a week inside a messy environment. If it feels weird, that's fine.
  2. Submitting locks the workspace. Once you submit (or the 7-day window closes — whichever comes first) the terminal, SQL runner, and apps become read-only. You can still browse what you found.
  3. If the deadline lands before you submit, we auto-submit whatever state you're in. No penalty for finishing early; no bonus for marathoning.
  4. Your token is single-use. If you accidentally delete the container or hit a hard install error, contact the assessment owner — we'll issue a replacement.
  5. If something looks broken, it might be. That's part of the exercise.
